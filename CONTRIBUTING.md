# Contributing to r-data

Thank you for your interest in contributing to the R Data Analysis Workspace!

## How to Contribute

### Reporting Issues
- Use GitHub Issues to report bugs or suggest features
- Provide clear descriptions and examples
- Include R version and package versions if reporting bugs

### Suggesting Improvements
- Example scripts for new analysis techniques
- Additional utility functions
- Improved documentation
- New R Markdown templates

### Submitting Changes
1. Fork the repository
2. Create a new branch for your feature
3. Make your changes
4. Test your changes thoroughly
5. Submit a pull request with a clear description

## Code Style Guidelines

- Follow the tidyverse style guide
- Use meaningful variable names
- Comment complex operations
- Include examples in documentation

## Adding New Features

When adding new features:
- Ensure they fit the repository's purpose (data analysis workspace)
- Include documentation and examples
- Update relevant README files
- Keep the structure organized

## Questions?

Feel free to open an issue for questions or discussions!
